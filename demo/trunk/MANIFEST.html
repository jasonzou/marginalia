Marginalia Demo Manifest

These are the implementation files that should be in this package:

db directory

These scripts initialize the database.

keywords.sql
Predefined keywords for the drop-down list presented while creating and editing annotations.
tables.sql
Annotation table definitions. Most applications will have table definitions very close to these.

util directory

These are useful scripts used to construct an application. They are not used at runtime.

locale2js.xsl
A sample conversion of locale/en_US.xml to produce www/strings.js, a Javascript file containing localized strings used by Marginalia. See the description of locale/en_US.xml in the manifest for the Marginalia library for details.

www directory

annotate-db.php
Database access code.
annotate.php
The service handler called when a request is made from any of the rest-*.js functions in Marginalia.
annotation.php
A class representing an annotation.
example-annotations.php
This can be used as a static source of annotations instead of the database back-end.
index.html
The application's main HTML page, with the content to be annotated. Includes extensive comments explaining how things work.
marginalia-config.js
Configuration settings needed by Marginalia.
marginalia-strings.js
The file generated by util/locale2js.xsl and included in the application to provide localized strings.
sample-config.php
An example of installation-specific configuration options. Should be edited and renamed to config.php.
static-annotate.js
A substitute for marginalia/rest-annotate.js which uses a static XML file instead of calling a back-end. Useful only for demo purposes.
static-prefs.js
A substitute for marginalia/rest-prefs.js which uses uses hard-coded preferences instead of calling a back end. Useful only for demo purposes.
word-range.php
Classes representing poinst or ranges in document content.
